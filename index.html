لماذا صياغة العقود في Word تحتاج مهارة خاصة؟

تأسيس الدافع — ٥ دقايق

العقد ليس مجرد نص — هو وثيقة قانونية ملزمة. أي خطأ في الترقيم أو التسلسل أو الصياغة قد يُفقده قيمته القانونية أو يُفسّره القضاء بطريقة مختلفة.

الحقيقة التي يعرفها كل محامٍ عقد منسق احترافياً بترقيم صحيح وبنود واضحة = قرينة على دقة محرّره. عقد مكتوب بأسطر متراكمة وترقيم يدوي = مستند هش يسهل الطعن فيه. الشكل في القانون جزء من المضمون.

04

حقول التعبئة (Content Controls)

تحويل العقد لنموذج ذكي — ٢٠ دقيقة
Content Controls عناصر تفاعلية تُدرج في المستند تسمح للمستخدم بملء بيانات محددة (نص، تاريخ، قائمة) دون تعديل النص القانوني الثابت. تُحوّل العقد لنموذج احترافي قابل للتعبئة.

أ. تفعيل تبويب Developer (المطوّر)

File → Options → Customize Ribbon → ☑ Developer

تبويب Developer لا يظهر افتراضياً — لازم تُفعّليه مرة واحدة فقط.

ب. أنواع Content Controls

📝

Rich Text Control

حقل نص يقبل أي محتوى بما فيه التنسيق. للأسماء والعناوين الطويلة.

🔤

Plain Text Control

حقل نص عادي بدون تنسيق. للأرقام القومية والمبالغ والأرقام.

📅

Date Picker Control

حقل تاريخ يعرض تقويماً للاختيار منه. لتاريخ العقد وتواريخ الاستحقاق.

🔽

Drop-Down List Control

قائمة منسدلة بخيارات محددة مسبقاً. لنوع العقد، المحافظة، طريقة السداد.

☑️

Check Box Control

خانة اختيار. لتأكيد الموافقة على شروط معينة في العقد.

🖼

Picture Control

حقل لإدراج صورة (صورة شخصية أو ختم رسمي) في مكان محدد.

ج. خطوات إدراج Content Control

Developer → اختاري نوع الـ Control المطلوب
الخطوةالإجراء
١ضعي المؤشر في مكان الحقل في العقد
٢Developer → اختاري نوع الـ Control
٣النقر على الـ Control → Properties (لضبط العنوان والنص الافتراضي)
٤في "Title": اكتبي اسم الحقل (مثل "اسم البائع")
٥في "Placeholder text": اكتبي التعليمات (مثل "اكتب اسمك الثلاثي هنا")
📋 مثال ٢ — عقد توكيل قابل للتعبئة

السيناريو: تحويل عقد توكيل ثابت إلى نموذج يُملأ بسهولة

  1. افتحي عقد التوكيل المكتوب كنص عادي
  2. احذفي اسم الموكّل — ضعي مكانه: Developer → Plain Text Control
  3. Properties → Title: "اسم الموكّل" | Placeholder: "اكتب الاسم كاملاً"
  4. احذفي تاريخ التوكيل — Developer → Date Picker Control
  5. Properties → Date Format: d/M/yyyy (التنسيق المصري)
  6. عند صلاحيات التوكيل: أضيفي Check Box Controls لكل صلاحية
  7. احفظي كـ Template: .dotx → "نموذج_عقد_توكيل.dotx"
  8. Review → Restrict Editing → Filling in forms فقط + Enforce Protection
النتيجة الموكّل يفتح النموذج → ينتقل بين الحقول بـ Tab → يملأ بياناته → يحفظ. لا يمكنه تعديل البنود القانونية أبداً.
تدريب ٢ — تحويل عقد إيجار لنموذج ذكي

أنشئي عقد إيجار قابل للتعبئة بـ Content Controls:

  1. فعّلي Developer Tab من: File → Options → Customize Ribbon
  2. أنشئي عقد إيجار بسيط (٥ بنود) مع النص القانوني الثابت
  3. أضيفي Plain Text Control لـ: اسم المالك | اسم المستأجر | عنوان الشقة
  4. أضيفي Date Picker Control لـ: تاريخ بداية الإيجار | تاريخ النهاية
  5. أضيفي Drop-Down List لـ "طريقة السداد": نقداً | تحويل بنكي | شيك
  6. أضيفي Check Box لـ "الموافقة على الشروط العامة"
  7. Review → Restrict Editing → Filling in forms → Enforce Protection
  8. احفظي كـ "نموذج_عقد_إيجار.dotx"

ب. الأرقام في العقود — قاعدة مهمة

القاعدة الذهبية للأرقام في العقود كل مبلغ مالي أو عدد في العقد يجب كتابته مرتين: بالأرقام أولاً ثم بالحروف بين قوسين. مثال: "مبلغ ٢٥٠,٠٠٠ جنيه (مئتان وخمسون ألف جنيه مصري)". هذا يمنع التزوير ويُعد شرطاً في كثير من أنواع العقود.

ج. Page Break بين الصفحات

كل بند رئيسي طويل يجب أن يبدأ من نفس الصفحة — لا يُقطع بين صفحتين:

حدّدي البند ← Home → Paragraph → Line and Page Breaks → ☑ Keep with next

د. ترقيم صفحات العقود

كل صفحة في العقد تحمل رقمها من إجمالي الصفحات لمنع إضافة أو حذف صفحات خفية:

Insert → Page Number → Bottom of Page → Page X of Y
📋 مثال ٣ — تنسيق عقد بيع احترافي خطوة بخطوة

السيناريو: عقد بيع شقة مكتوب كنص عشوائي — المطلوب تحويله لعقد احترافي

  1. الصفحة: A4 — هوامش: يمين 3 سم (للتجليد)، باقي الجهات 2.5 سم
  2. البسملة والعنوان: Sakkal Majalla 18 Bold Center Underline
  3. بيانات الأطراف: جدول 1×2 بلا حدود — البائع يمين، المشتري يسار
  4. البنود: Multilevel List → ترقيم تلقائي ١، ١.١، ١.١.١
  5. نص البنود: Sakkal Majalla 12 — Justify — تباعد 1.5
  6. الأرقام المالية: أضيفي الأرقام بالحروف بين قوسين بعد كل مبلغ
  7. Keep with next: فعّليه على عناوين البنود لمنع قطعها
  8. Footer: "صفحة X من Y — عقد البيع المحرر في ..."
  9. Content Controls: أضيفيها للبيانات المتغيرة (الأسماء، التواريخ، المبالغ)
  10. الحماية: Restrict Editing + Filling in forms + كلمة مرور
06

البنود القياسية في العقود المصرية

محتوى قانوني جاهز للاستخدام — ١٠ دقايق

هناك بنود ثابتة تقريباً في كل عقد مصري. المحامي المحترف يحفظها في مكتبة نصوص (Building Blocks) لاستخدامها الفوري.

⚖️

بند الاختصاص القضائي

"يختص القضاء المصري بنظر أي نزاع ينشأ عن تفسير أو تنفيذ هذا العقد، وتختص بنظره محاكم [المحافظة]."

📅

بند القوة القاهرة

"لا يُعدّ أي طرف مسؤولاً عن التأخير في التنفيذ إذا كان ناجماً عن ظروف قاهرة خارجة عن إرادته..."

🔔

بند الإخطار

"يُعدّ الإخطار صحيحاً إذا أُرسل بخطاب مسجَّل مصحوب بعلم الوصول أو بالبريد الإلكتروني المعتمد..."

🔒

بند السرية

"يلتزم الأطراف بالحفاظ على سرية جميع المعلومات المتعلقة بهذا العقد وعدم الإفصاح عنها..."

✍️

بند التعديل

"لا يكون أي تعديل في بنود هذا العقد صحيحاً وملزماً إلا إذا تمّ بموافقة خطية من الطرفين..."

📋

بند الحلول

"يسري هذا العقد على الخلفاء والورثة والمنتقل إليهم من الطرفين..."

بناء مكتبة البنود القانونية احفظي كل بند قياسي كـ Quick Part (Building Block): حدّدي البند ← Insert → Quick Parts → Save Selection. ثم في أي عقد جديد: اكتبي اسم البند واضغطي F3 — البند يظهر كاملاً في ثانية واحدة.
تدريب ٣ — بناء مكتبة البنود القانونية

أنشئي مكتبة بنود قانونية جاهزة للاستخدام:

  1. أنشئي مستند جديد واكتبي البنود الستة القياسية أعلاه بتنسيق احترافي
  2. حدّدي بند "الاختصاص القضائي" كاملاً
  3. Insert → Quick Parts → Save Selection to Quick Part Gallery
  4. Name: "اختصاص" | Gallery: AutoText | OK
  5. كرّري لكل البنود الستة (اختصاص / قوة_قاهرة / إخطار / سرية / تعديل / حلول)
  6. افتحي مستند عقد جديد — اكتبي "اختصاص" واضغطي F3 — البند يظهر كاملاً
  7. جرّبي استدعاء كل البنود وتجميعها في عقد واحد بضغطات فقط

رمز المادة القانونية (Section Sign). Insert → Symbol. يُستخدم للإشارة لمواد القانون المصري.

¶

رمز الفقرة (Pilcrow). يظهر عند تفعيل Show/Hide. يساعد اكتشاف مشاكل التنسيق الخفية.

{ }

أقواس الحقول — تظهر مع Alt+F9. تشير لوجود حقل تلقائي (Content Control أو Mail Merge).

08

قاموس مصطلحات المحاضرة

مرجع سريع للمراجعة
Multilevel List
قائمة متعددة المستويات — نظام ترقيم تلقائي هرمي (١ ← ١.١ ← ١.١.١) يتحدث تلقائياً عند الإضافة أو الحذف.
Content Control
حقل تعبئة ذكي يُدرج في العقد — يسمح بملء البيانات دون تعديل النص القانوني الثابت.
Developer Tab
تبويب المطوّر — يحتوي أدوات Content Controls وحماية النماذج. يجب تفعيله يدوياً من File → Options.
Plain Text Control
حقل نص عادي بلا تنسيق. للأسماء والأرقام القومية والمبالغ في العقود.
Date Picker Control
حقل تاريخ يعرض تقويماً. لتاريخ العقد وتواريخ الاستحقاق.
Drop-Down List Control
قائمة منسدلة بخيارات محددة مسبقاً. لطريقة السداد أو نوع العقد.
Check Box Control
خانة اختيار تفاعلية. للموافقة على بنود أو تحديد خيارات في العقد.
Filling in Forms
خيار الحماية الذي يسمح فقط بملء حقول التعبئة — ويمنع تعديل أي نص آخر.
Keep with Next
خيار تنسيق يمنع فصل الفقرة عن الفقرة التالية — يضمن بقاء عنوان البند مع أول سطر من نصه.
Building Blocks
مكتبة نصوص قانونية جاهزة محفوظة — تُستدعى بكلمة مفتاحية + F3.
Quick Parts
نفس Building Blocks — القائمة التي تُدار منها النصوص الجاهزة من Insert.
Page X of Y
ترقيم صفحات يعرض رقم الصفحة من إجمالي الصفحات — لمنع إضافة/حذف صفحات خفية في العقد.

التدريب الختامي الشامل

بناء عقد بيع احترافي كامل — ١٥ دقيقة
تدريب ختامي — عقد بيع شقة احترافي قابل للتعبئة

المطلوب: إنشاء عقد بيع شقة سكنية كامل ومؤمَّن بـ Content Controls

  1. الإعداد: مستند جديد — A4 — هامش يمين 3 سم — باقي الجهات 2.5 سم
  2. الرأس: بسملة (16 Bold Center) + "عقد بيع شقة سكنية" (18 Bold Center Underline)
  3. الأطراف: جدول 1×2 بلا حدود — Plain Text Controls لـ (اسم البائع، الرقم القومي، العنوان) في كل خلية
  4. ترقيم البنود: Multilevel List لـ ٧ بنود رئيسية مع فقراتها الفرعية:
    • البند ١: تمهيد وسند الملكية
    • البند ٢: محل العقد (Plain Text للعنوان والمساحة)
    • البند ٣: الثمن والسداد (Plain Text للمبلغ + Check Box لطريقة السداد)
    • البند ٤: التسليم (Date Picker لتاريخ التسليم)
    • البند ٥: التزامات البائع
    • البند ٦: الفسخ والغرامة
    • البند ٧: أحكام عامة (استخدمي Building Blocks للاختصاص القضائي والقوة القاهرة)
  5. Footer: "صفحة X من Y — عقد البيع المحرر في " + Date Picker
  6. الحماية: Developer → Restrict Editing → Filling in forms → Enforce Protection بكلمة مرور "law2025"
  7. الحفظ: Save As → Word Template (.dotx) → "عقد_بيع_شقة_سكنية.dotx"

الواجب البيتي والمراجعة الذاتية

سلّميه في المحاضرة القادمة

الواجب: بناء مكتبة عقود قانونية متكاملة

  • عقد إيجار .dotx: عقد إيجار سكني كامل (٨ بنود على الأقل) — مع Multilevel List للبنود + Content Controls للبيانات المتغيرة + حماية بكلمة مرور.
  • عقد توكيل .dotx: عقد توكيل عام — مع Drop-Down List لنوع التوكيل (عام/خاص) + Check Box للصلاحيات + Date Picker للمدة.
  • مكتبة البنود: احفظي البنود الستة القياسية (الاختصاص، القوة القاهرة، الإخطار، السرية، التعديل، الحلول) كـ Building Blocks — وسلّمي لقطة شاشة تُظهر المكتبة.
  • الأرقام بالحروف: في كل مبلغ مالي في العقدين — اكتبيه بالأرقام ثم بين قوسين بالحروف كاملاً.
  • Footer احترافي: كل صفحة تحمل "صفحة X من Y" مع عنوان العقد — بدون ظهور في الغلاف.
